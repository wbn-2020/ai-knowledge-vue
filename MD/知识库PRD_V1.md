# KnowFlow AI：个人知识库与智能文档问答平台 PRD 功能需求清单

> 项目定位：面向个人用户/小团队的知识库管理与智能文档问答平台。用户可以上传 PDF、Word、Markdown、TXT 等文档，系统自动解析、切分、向量化，用户可基于个人知识库进行智能问答、语义检索、文档摘要和知识整理。

---

## 1. 项目目标

### 1.1 产品目标

- 帮助用户集中管理个人学习资料、项目文档、技术笔记和业务资料。
- 支持基于文档内容的 AI 问答，降低查找资料和理解文档的成本。
- 提供知识库、文档、问答记录、标签、权限等完整管理能力。
- 作为 Java 全栈个人作品集项目，重点体现：
  - Spring Boot 后端开发能力；
  - 文件上传与异步处理能力；
  - AI 大模型接入能力；
  - 向量检索与 RAG 应用能力；
  - RBAC 权限、后台管理、日志审计等工程能力；
  - Vue 前端交互与管理后台能力。

### 1.2 第一版 MVP 目标

第一版重点完成“可上传文档、可创建知识库、可基于知识库问答、可管理用户和文档”的最小闭环。

MVP 闭环：

```text
注册/登录
  -> 创建知识库
    -> 上传文档
      -> 文档解析与向量化
        -> 基于知识库提问
          -> 查看 AI 回答和引用来源
            -> 管理知识库/文档/问答记录
```

---

## 2. 用户角色划分

### 2.1 前台普通用户

普通用户是平台的主要使用者，负责创建个人知识库、上传文档、发起智能问答、管理问答记录。

### 2.2 后台管理员

管理员负责平台运营管理，包括用户管理、知识库管理、文档管理、AI 配置、系统参数、数据统计和日志审计。

### 2.3 可选扩展角色

- 团队成员：后续支持团队知识库协作。
- 审核员：后续支持公开知识库、共享文档审核。
- 超级管理员：后续用于多租户或 SaaS 版本管理。

---

## 3. 功能优先级说明

- 【核心 MVP 功能】：第一版必须实现，保证项目可完整演示和部署。
- 【扩展功能】：后续版本实现，用于提升项目完整度和面试亮点。

# 4. 前台功能需求清单：普通用户端

## 4.1 用户认证与账号模块

### 【核心 MVP 功能】

#### 4.1.1 用户注册

- 支持手机号或邮箱注册。
- 支持用户名、密码、确认密码输入。
- 校验用户名是否重复。
- 校验密码复杂度。
- 注册成功后自动创建默认用户空间。

#### 4.1.2 用户登录

- 支持用户名/邮箱 + 密码登录。
- 登录成功后返回访问令牌。
- 支持登录态保持。
- 支持登录失败提示。
- 支持账号禁用状态拦截。

#### 4.1.3 退出登录

- 用户可主动退出当前账号。
- 清除本地登录态。
- 返回登录页。

#### 4.1.4 当前用户信息

- 查看当前登录用户的基础信息。
- 展示用户名、头像、邮箱、注册时间、账号状态。

#### 4.1.5 修改个人资料

- 修改昵称。
- 修改头像。
- 修改邮箱。
- 修改个人简介。

#### 4.1.6 修改密码

- 输入旧密码。
- 输入新密码。
- 确认新密码。
- 修改成功后要求重新登录。

### 【扩展功能】

#### 4.1.7 第三方登录

- 支持 GitHub 登录。
- 支持 Google 登录。
- 支持微信扫码登录。

#### 4.1.8 忘记密码

- 邮箱验证码找回密码。
- 手机验证码找回密码。

#### 4.1.9 账号安全

- 登录设备管理。
- 登录历史查看。
- 异地登录提醒。
- 二次验证。

## 4.3 知识库管理模块

### 【核心 MVP 功能】

#### 4.3.1 创建知识库

- 输入知识库名称。
- 输入知识库简介。
- 选择知识库图标。
- 选择知识库类型：技术学习、项目资料、产品文档、论文资料、其他。
- 创建成功后进入知识库详情页。

#### 4.3.2 知识库列表

- 展示当前用户创建的所有知识库。
- 展示知识库名称、简介、文档数量、创建时间、更新时间。
- 支持按创建时间排序。
- 支持按更新时间排序。
- 支持关键词搜索知识库。

#### 4.3.3 查看知识库详情

- 查看知识库基础信息。
- 查看知识库内文档列表。
- 查看知识库最近问答记录。
- 查看知识库处理状态。

#### 4.3.4 编辑知识库

- 修改知识库名称。
- 修改知识库简介。
- 修改知识库图标。
- 修改知识库分类。

#### 4.3.5 删除知识库

- 删除前二次确认。
- 删除知识库时同步处理关联文档记录。
- 删除知识库时同步处理向量索引数据。
- 删除后不可恢复，MVP 可采用物理删除或逻辑删除，建议逻辑删除。

#### 4.3.6 知识库状态管理

- 正常。
- 处理中。
- 解析失败。
- 已禁用。

#### 4.3.7 知识库权限基础控制

- MVP 阶段默认知识库仅自己可见。
- 用户只能访问自己创建的知识库。
- 后端接口必须校验知识库归属关系。

### 【扩展功能】

#### 4.3.8 知识库分享

- 生成分享链接。
- 设置分享有效期。
- 设置是否允许他人提问。
- 设置是否允许他人查看原文。

#### 4.3.9 知识库协作

- 邀请成员加入知识库。
- 设置成员角色：只读、可编辑、管理员。
- 移除成员。
- 查看成员操作记录。

#### 4.3.10 知识库模板

- 技术学习模板。
- 项目文档模板。
- 论文阅读模板。
- 面试资料模板。

#### 4.3.11 知识库归档

- 归档不常用知识库。
- 归档后不出现在默认列表中。
- 支持恢复归档。

## 4.5 文档解析与知识索引模块

### 【核心 MVP 功能】

#### 4.5.1 文档文本解析

- PDF 文本提取。
- DOCX 文本提取。
- TXT 内容读取。
- Markdown 内容读取。
- 提取失败时记录失败原因。

#### 4.5.2 文本切片

- 按固定长度切分文本。

- 记录切片顺序。
- 记录切片所属文档。

- 支持手动刷新状态。

### 【扩展功能】

- 支持图片中的文字识别。

#### 4.5.6 表格解析

- 提取 PDF 表格。

- 保留表格结构。

#### 4.5.7 文档结构化解析

- 自动识别标题层级。
- 自动生成目录。
- 自动提取关键词。
- 自动提取摘要。

#### 4.5.8 索引重建

- 单文档重建索引。

## 4.6 智能问答模块

### 【核心 MVP 功能】

#### 4.6.1 选择知识库提问

- 用户选择一个知识库。
- 在知识库范围内发起问题。
- 问题不能为空。
- 问题长度限制。

#### 4.6.2 AI 回答生成

- 根据用户问题进行语义检索。
- 从知识库中召回相关文档切片。
- 将召回内容组装成提示词。
- 调用大模型生成回答。
- 返回回答内容。

#### 4.6.3 引用来源展示

- 展示回答引用的文档名称。
- 展示引用片段摘要。
- 展示引用片段相似度。
- 支持点击查看引用详情。

#### 4.6.4 问答会话

- 每次问答保存为一条会话记录。
- 支持连续追问。
- 保留上下文消息。
- 支持查看历史消息。

#### 4.6.5 新建会话

- 用户可在指定知识库下新建会话。
- 新会话默认标题可由首个问题生成。
- 会话与知识库绑定。

#### 4.6.6 清空当前会话

- 清空当前会话消息。
- 删除前二次确认。

#### 4.6.7 删除会话

- 删除指定问答会话。
- 删除会话下所有消息。

#### 4.6.8 回答状态展示

- 正在思考。
- 正在检索文档。
- 正在生成回答。
- 生成完成。
- 生成失败。

### 【扩展功能】

#### 4.6.9 流式输出

- AI 回答逐字输出。
- 支持中断生成。
- 支持重新生成。

#### 4.6.10 多知识库问答

- 用户可同时选择多个知识库。
- 系统在多个知识库中检索。
- 回答中区分引用来源。

#### 4.6.11 指定文档问答

- 用户选择某个文档进行问答。
- 检索范围限制在指定文档内。

#### 4.6.12 回答反馈

- 点踩回答。
- 标记回答不准确。

- 精准回答模式。

# 9. 关键业务流程

## 9.1 用户上传文档并问答流程

```text
用户登录
  -> 创建知识库
    -> 上传文档
      -> 后端保存文件
        -> 创建文档记录
          -> 创建解析任务
            -> 解析文本
              -> 文本切片
                -> 调用 Embedding 模型
                  -> 保存向量索引
                    -> 文档状态变为可问答
                      -> 用户选择知识库提问
                        -> 系统向量检索相关片段
                          -> 拼接 Prompt
                            -> 调用大模型
                              -> 返回回答和引用来源
                                -> 保存问答记录
```

## 9.2 文档解析失败重试流程

```text
文档解析失败
  -> 用户或管理员查看失败原因
    -> 点击重新解析
      -> 清理旧的部分解析数据
        -> 重新创建解析任务
          -> 解析成功
            -> 重新生成向量索引
              -> 文档恢复可问答状态
```

## 9.3 管理员禁用用户流程

```text
管理员登录后台
  -> 进入用户管理
    -> 搜索目标用户
      -> 查看用户详情
        -> 点击禁用用户
          -> 填写禁用原因
            -> 确认禁用
              -> 用户账号状态变更为禁用
                -> 用户无法继续登录
                  -> 记录管理员操作日志
```

# 11. 非功能需求

## 11.1 性能需求

- 文档上传接口需要支持大文件上传，MVP 阶段建议单文件限制 20MB。
- 文档解析需要异步执行，避免阻塞上传接口。
- 普通问答接口响应时间建议控制在 3-15 秒内，具体取决于模型响应速度。
- 知识库列表、文档列表、问答历史必须分页。
- 后台列表类查询必须支持分页和条件筛选。

## 11.2 安全需求

- 用户只能访问自己的知识库和文档。
- 后台接口必须校验管理员权限。
- 文件上传必须限制类型和大小。
- 文件路径不能直接暴露真实服务器路径。
- API Key 必须加密或安全存储。
- 操作日志需要记录关键操作。

## 11.3 可用性需求

- 文档解析失败需要展示明确原因。
- AI 调用失败需要给出友好提示。
- 删除类操作需要二次确认。
- 长耗时操作需要展示状态。
- 空列表页面需要有引导文案。

## 11.4 可维护性需求

- AI 模型调用需要抽象统一接口，便于替换供应商。
- 文档解析逻辑需要按文件类型拆分策略。
- 向量数据库访问需要封装 Repository 或 Service。
- Prompt 模板需要配置化。
- 日志、异常、鉴权需要统一处理。
