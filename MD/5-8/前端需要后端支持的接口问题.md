# 前端需要后端支持的接口问题

更新时间：2026-05-08

本文件记录本次用户端 MVP 主链路改造中，前端已经按页面需求传参或预留交互，但需要后端明确支持的接口契约。

## 1. 文档分页列表筛选参数

相关前端文件：

- `src/api/knowledge.ts`
- `src/views/app/DocumentManageView.vue`
- `src/views/app/KnowledgeBaseDetailView.vue`

当前前端调用：

```text
GET /api/documents
```

前端已传入的查询参数：

- `knowledgeBaseId`
- `keyword`
- `parseStatus`
- `embeddingStatus`
- `pageNo`
- `pageSize`

需要后端支持：

1. `knowledgeBaseId`：只返回当前用户在指定知识库下的文档。
2. `parseStatus`：按 `PENDING / PARSING / SUCCESS / FAILED` 过滤。
3. `embeddingStatus`：按 `PENDING / PROCESSING / SUCCESS / FAILED` 过滤。
4. 所有筛选条件必须与分页同时生效，不能先分页后过滤。
5. 返回格式保持：

```json
{
  "list": [],
  "total": 0,
  "pageNo": 1,
  "pageSize": 10
}
```

## 2. 问答过程状态

相关前端文件：

- `src/views/app/ChatView.vue`

当前前端调用：

```text
POST /api/chat/ask
```

当前前端只能在单次请求前后展示粗粒度状态：

- 检索中
- 生成中
- 完成
- 失败

如后端后续希望展示更准确的 RAG 阶段状态，需要补充以下任一能力：

1. 在 `/chat/ask` 响应中返回阶段耗时或状态明细，例如 `retrievalElapsedMs`、`generationElapsedMs`。
2. 或提供任务式问答状态查询接口。
3. 或提供流式/SSE 状态事件。

MVP 当前不强依赖该增强接口，前端已用同步请求状态满足基础演示。

## 3. 后台知识库列表筛选参数

相关前端文件：

- `src/api/knowledge.ts`
- `src/views/admin/AdminKnowledgeView.vue`

当前前端调用：

```text
GET /api/admin/knowledge-bases
```

前端已传入的查询参数：

- `keyword`
- `status`
- `pageNo`
- `pageSize`

需要后端支持：

1. `keyword`：按知识库名称、简介或创建人进行模糊搜索。
2. `status`：按 `NORMAL / PROCESSING / FAILED / DISABLED` 过滤。
3. 筛选必须与分页同时生效。

## 4. 后台文档列表筛选参数

相关前端文件：

- `src/api/knowledge.ts`
- `src/views/admin/AdminDocumentView.vue`
- `src/views/admin/KnowledgeDetailView.vue`

当前前端调用：

```text
GET /api/admin/documents
```

前端已传入的查询参数：

- `keyword`
- `knowledgeBaseId`
- `parseStatus`
- `pageNo`
- `pageSize`

需要后端支持：

1. `knowledgeBaseId`：后台知识库详情必须只返回当前知识库关联文档，不能返回全平台文档后由前端过滤。
2. `parseStatus`：按 `PENDING / PARSING / SUCCESS / FAILED` 过滤。
3. `keyword`：按文档名称或原始文件名搜索。
4. 响应文档项建议返回 `errorMessage` 或 `failReason`，用于展示失败原因。

## 5. 后台任务列表筛选与重试接口

相关前端文件：

- `src/api/knowledge.ts`
- `src/views/admin/TaskView.vue`

当前前端调用：

```text
GET /api/admin/document-tasks
POST /api/admin/document-tasks/{id}/retry
```

前端已传入的查询参数：

- `status`
- `taskType`
- `keyword`
- `pageNo`
- `pageSize`

需要后端支持：

1. `status`：按 `PENDING / PROCESSING / SUCCESS / FAILED` 过滤。
2. `taskType`：按 `PARSE / CHUNK / EMBEDDING / INDEX` 过滤，若后端枚举不同，需要统一契约。
3. `keyword`：按文档名称或任务信息搜索。
4. 失败任务返回 `reason`、`errorMessage` 或 `failReason`。
5. `POST /admin/document-tasks/{id}/retry` 用于后台重试失败任务。

## 6. 后台模型配置字段契约

相关前端文件：

- `src/api/knowledge.ts`
- `src/views/admin/ModelConfigView.vue`

当前前端调用：

```text
GET /api/admin/config/models
POST /api/admin/config/models
PUT /api/admin/config/models/{id}
POST /api/admin/config/models/{id}/test
```

前端提交字段：

- `name`
- `provider`
- `modelName`
- `model`
- `endpoint`
- `apiKey`
- `modelType`
- `enabled`
- `temperature`

需要后端支持：

1. `modelType`：区分 `LLM` 和 `EMBEDDING`。
2. `modelName`：作为模型名称主字段；当前前端也兼容提交 `model`。
3. `apiKey`：新增时必填，编辑时留空表示不修改。
4. 查询列表时不得返回明文 API Key，只返回 `apiKeyMasked` 或 `maskedApiKey`。
5. 测试连接接口应根据模型类型执行 LLM 或 Embedding 的最小可用性测试。
