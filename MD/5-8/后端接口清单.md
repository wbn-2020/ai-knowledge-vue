# KnowFlow AI 后端接口清单

更新时间：2026-05-08

## 1. 全局约定

### 1.1 基础地址

后端默认端口和上下文：

```text
http://localhost:8080/api
```

下文接口路径均省略 `/api` 前缀。

### 1.2 鉴权

除以下接口外，其余接口都需要登录：

```text
POST /auth/register
POST /auth/login
POST /admin/auth/login
```

登录后前端在请求头携带：

```http
Authorization: Bearer <token>
```

后台管理接口需要管理员账号，路径通常以 `/admin` 开头。

### 1.3 统一响应

普通 JSON 接口统一返回：

```json
{
  "code": 0,
  "message": "ok",
  "data": {}
}
```

分页数据结构：

```json
{
  "list": [],
  "total": 0,
  "pageNo": 1,
  "pageSize": 10
}
```

文件下载/导出接口返回二进制或文本文件，不包 `ApiResponse`。

### 1.4 常用枚举

```text
UserStatus: ENABLED, DISABLED
UserRole: USER, ADMIN
KnowledgeBaseStatus: NORMAL, PROCESSING, FAILED, DISABLED
DocumentParseStatus: PENDING, PARSING, SUCCESS, FAILED
EmbeddingStatus: PENDING, PROCESSING, SUCCESS, FAILED
TaskStatus: PENDING, PROCESSING, SUCCESS, FAILED
MessageRole: USER, ASSISTANT
```

## 2. 认证与用户

### 2.1 用户注册

```http
POST /auth/register
Content-Type: application/json
```

请求体：

```json
{
  "username": "demo",
  "email": "demo@knowflow.local",
  "password": "admin123",
  "confirmPassword": "admin123"
}
```

说明：
- 注册成功后自动创建默认知识库。
- 返回 token 和用户信息。

响应 data：

```json
{
  "token": "jwt-token",
  "user": {
    "id": 1,
    "username": "demo",
    "email": "demo@knowflow.local",
    "nickname": "demo",
    "avatar": null,
    "bio": null,
    "status": "ENABLED",
    "role": "USER",
    "createTime": "2026-05-08T14:00:00"
  }
}
```

### 2.2 用户登录

```http
POST /auth/login
Content-Type: application/json
```

请求体：

```json
{
  "account": "demo",
  "password": "admin123"
}
```

响应同注册。

### 2.3 退出登录

```http
DELETE /auth/logout
Authorization: Bearer <token>
```

说明：
- 后端为无状态 JWT，接口返回成功。
- 前端需要主动清除本地 token。

### 2.4 当前用户信息

```http
GET /users/me
Authorization: Bearer <token>
```

响应 data：`UserVO`

### 2.5 修改个人资料

```http
PUT /users/me
Authorization: Bearer <token>

```

请求体：

```json
{
  "nickname": "Demo User",

  "avatar": "https://example.com/avatar.png",
  "bio": "个人简介"
}
```

响应 data：`UserVO`

### 2.6 修改密码

```http
PUT /users/me/password

```

请求体：

```json
{
  "oldPassword": "admin123",
  "newPassword": "newpass123",
  "confirmPassword": "newpass123"
```

## 4. 知识库

### 4.1 创建知识库

```http
POST /knowledge-bases
Authorization: Bearer <token>
Content-Type: application/json
```

请求体：

```json
{
  "name": "Spring Boot Notes",
  "description": "后端学习资料",
  "icon": "book",
  "category": "backend"
}
```

响应 data：`KnowledgeBaseVO`

### 4.2 知识库分页列表

```http
GET /knowledge-bases?keyword=&pageNo=1&pageSize=10&sortBy=updateTime
Authorization: Bearer <token>
```

参数：

| 参数 | 类型 | 必填 | 说明 |
|---|---|---|---|
| keyword | string | 否 | 名称关键词 |
| pageNo | number | 否 | 默认 1 |
| pageSize | number | 否 | 默认 10 |
| sortBy | string | 否 | `updateTime` 或 `createTime` |

响应 data：`PageResponse<KnowledgeBaseVO>`

### 4.3 知识库基础详情

```http
GET /knowledge-bases/{id}
Authorization: Bearer <token>
```

响应 data：`KnowledgeBaseVO`

### 4.4 知识库聚合详情

```http
GET /knowledge-bases/{id}/detail
Authorization: Bearer <token>
```

响应 data：

```json
{
  "knowledgeBase": {},
  "recentDocuments": [],
  "recentSessions": [],
  "processingStatus": "NORMAL"
}
```

### 4.5 编辑知识库

```http
PUT /knowledge-bases/{id}
Authorization: Bearer <token>
Content-Type: application/json
```

请求体同创建知识库。

### 4.6 删除知识库

```http
DELETE /knowledge-bases/{id}
Authorization: Bearer <token>
```

说明：
- 会级联逻辑删除知识库下文档、切片、处理任务和会话。

## 15. 前端注意事项

- 所有业务请求都要加 `Authorization`，文件下载和导出也需要。
- 上传文档使用 `multipart/form-data`，不要手动设置错误的 `Content-Type` boundary。
- 后台接口需使用管理员 token。
- DeepSeek API Key 不应出现在前端，由后端配置或环境变量管理。
- PDF/Word/CSV/Markdown 下载接口不要按统一 JSON 解析。
